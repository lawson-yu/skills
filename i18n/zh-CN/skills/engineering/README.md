# Engineering

我每天用于代码工作的技能。

- **[diagnose](./diagnose/SKILL.md)** — 用于棘手 bug 和性能回退的纪律化诊断循环：reproduce → minimise → hypothesise → instrument → fix → regression-test。
- **[grill-with-docs](./grill-with-docs/SKILL.md)** — 通过质询式讨论让你的计划接受现有领域模型的检验，打磨术语，并内联更新 `CONTEXT.md` 和 ADR。
- **[triage](./triage/SKILL.md)** — 通过一套分诊角色状态机来处理问题。
- **[improve-codebase-architecture](./improve-codebase-architecture/SKILL.md)** — 在 `CONTEXT.md` 的领域语言与 `docs/adr/` 中决策的指导下，发现代码库中可深化架构的机会。
- **[setup-matt-pocock-skills](./setup-matt-pocock-skills/SKILL.md)** — 搭建仓库级配置（issue tracker、triage 标签词汇、领域文档布局），供其他 engineering 技能使用。
- **[tdd](./tdd/SKILL.md)** — 采用 red-green-refactor 循环进行测试驱动开发。一次一个纵向切片地构建功能或修复 bug。
- **[to-issues](./to-issues/SKILL.md)** — 使用纵向切片将任意计划、spec 或 PRD 拆分为可独立领取的 GitHub issues。
- **[to-prd](./to-prd/SKILL.md)** — 将当前对话上下文转换为 PRD，并作为 GitHub issue 提交。
- **[zoom-out](./zoom-out/SKILL.md)** — 让 agent 跳出细节，针对不熟悉的代码区域提供更广的上下文或更高层级的视角。
